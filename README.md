1. 项目背景与目标 (Project Overview)
1.1 背景

大多数理财App过于侧重“记账”和“数字”，忽视了《小狗钱钱》中强调的核心理念：自信心（成功日记）、目标可视化（梦想相册）与执行力（72小时法则）。用户需要一款结合“心理建设”与“轻量化理财”的工具。

1.2 核心目标

打造一款帮助用户建立“富人思维”的个人成长App。通过数字化《小狗钱钱》的方法论，提升用户的自信心和执行力，并养成储蓄习惯。

1.3 用户画像

初级理财小白：想存钱但控制不住消费，对枯燥的记账反感。

自我提升者：缺乏自信，做事拖延，需要正向反馈机制。

2. 功能架构图 (Information Architecture)

App 采用底部导航栏（Bottom Navigation）结构，共分四个模块：

首页 (Home/Goose)：金鹅资产概览 + 每日一句（语录）。

日记 (Journal)：成功日记 + 历史回顾。

梦想 (Dreams)：梦想相册 + 储蓄罐进度。

行动 (Action)：72小时任务清单。

3. 详细功能需求 (Functional Requirements)
模块一：成功日记 (The Success Journal)

优先级：P0 (最高)

用户故事：
作为用户，我希望每天记录5件成功的小事，以便在我不自信时回顾，获得力量。

功能细节：

每日记录流：

界面展示5个空的卡片插槽。

支持文本输入（MVP）、语音转文字（后续迭代）。

彩蛋逻辑： 当用户填满5条时，播放“金币掉落”音效或“小狗欢呼”动画。

日历视图/列表视图：

按日期查看历史日记。

自信模式（Filter）： 提供一个“心情低落”按钮，点击后随机展示过去的高光时刻卡片。

验收标准 (Acceptance Criteria)：

能成功创建、编辑、删除当天的日记。

填完5条后，UI有明显的完成状态反馈（如打钩动画）。

数据本地持久化存储。

模块二：梦想相册与储蓄罐 (Dream Vision & Jars)

优先级：P0 (最高)

用户故事：
作为用户，我希望上传我想买的东西的照片，并记录我为它存了多少钱，以便直观地看到距离目标还有多远。

功能细节：

创建梦想：

必填：梦想名称、目标金额。

必选：上传封面图（视觉刺激）。

储蓄操作：

点击某个梦想卡片，选择“投喂/存入”。

输入金额。

视觉反馈： 进度条增长，若存满，封面图从黑白变彩色（或增加金边特效）。

梦想墙：

展示所有“进行中”和“已达成”的梦想。

验收标准 (Acceptance Criteria)：

图片上传/裁剪功能正常。

存钱进度条计算准确（当前/目标 %）。

达成目标后，梦想卡片自动归档至“成就殿堂”。

模块三：72小时行动法则 (The 72-Hour Rule)

优先级：P1 (高)

用户故事：
作为用户，当我产生一个新想法时，我希望开启一个倒计时，提醒我在3天内迈出第一步，否则任务失效。

功能细节：

新建任务：

输入任务名称（如：开通基金账户）。

创建时自动记录当前时间戳。

倒计时状态机：

0-24小时： 绿色倒计时，显示“最佳行动期”。

24-48小时： 黄色倒计时，显示“警告”。

48-72小时： 红色倒计时，显示“最后机会”。

>72小时： 任务自动变灰，标记为“已失效/错失”，不可再勾选完成。

任务操作：

标记完成（Done）。

放弃（Give up）。

验收标准 (Acceptance Criteria)：

倒计时准确，且App后台运行/关闭后重开时间计算无误。

超过72小时的任务被强制锁定，无法点击完成。

模块四：金鹅资产管理 (The Golden Goose)

优先级：P1 (中)

用户故事：
作为用户，我想区分“养鹅的钱”（本金）和“零花钱”，防止我杀鸡取卵。

功能细节：

资产概览：

显示一只鹅（代表不可动用的本金总额）。

显示一篮金蛋（代表利息/可分配收入）。

分配逻辑（简化版）：

记一笔收入时，系统弹窗询问：“多少比例喂给鹅？”（默认建议50%）。

根据比例，自动将金额加在“鹅”或“金蛋”的数值上。

4. 非功能性需求 (Non-functional Requirements)

数据隐私：MVP版本采用 Local-First 策略（SQLite 或 Realm），数据仅存储在用户手机本地，不上传服务器，保护隐私。

性能：应用冷启动时间需 < 1.5秒。

UI/UX 风格：

色调：暖金色、橙色、米白色（财富与温馨感）。

吉祥物：需要设计一个简单的“拉布拉多”图标或插画作为引导助手。

5. 数据埋点与指标 (Analytics) - 可选，用于后期优化

核心指标：日记完成率（每天写满5条的用户占比）。

转化指标：梦想达成数。

流失预警：72小时任务失效比例（如果太高，说明用户行动力受阻，需增加提醒）。

6. 开发路线图 (Roadmap)
Phase 1: MVP (预计周期：2-3周)

周1：搭建项目框架，完成底部导航，实现“成功日记”的增删改查。

周2：实现“梦想相册”及存钱逻辑，图片本地存储。

周3：实现“72小时”倒计时逻辑，UI美化，打包测试。

Phase 2: 迭代 (预计周期：TBD)

增加 iCloud/Google Drive 数据备份。

增加 AI 助手对话功能（模仿钱钱的语气点评日记）。

增加桌面小组件 (Widget) 查看倒计时和梦想进度。
